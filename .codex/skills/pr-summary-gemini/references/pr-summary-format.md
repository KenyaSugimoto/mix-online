# PR Summary Format

このSkillでGeminiに生成させるPR本文の必須構造。

## 必須見出し（固定順）

1. `## 変更概要`
2. `## 背景・目的`
3. `## 変更点`
4. `## テスト結果`
5. `## 影響範囲`
6. `## 未対応項目`
7. `## 関連ドキュメント`
8. `## レビュー観点`

## 記述ルール

- 日本語で簡潔に書く
- 差分・コミット・実行ログにない情報を断定しない
- 未実行項目は `未実行`、未提供情報は `未提供` と明記する
- 影響範囲は API / DB / UI / Realtime の観点を含める
- レビュー観点は仕様整合・異常系・テスト妥当性を優先する
