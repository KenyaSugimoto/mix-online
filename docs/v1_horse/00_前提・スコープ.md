## 2. 背景

MVP は Stud 3種（`STUD_HI`, `RAZZ`, `STUD_8`）を対象に開発中である。
v1 では HORSE（`HOLD_EM`, `OMAHA_8`, `RAZZ`, `STUD_HI`, `STUD_8`）を対象に拡張する。

## 3. 固定済み前提（合意事項）

- v1 対象ゲームは HORSE 5種。
- ローテーションは 6 ハンド単位で切替。
- v1 は `FIXED_LIMIT` のみを対象。
- API/Realtime 契約は「MVP互換を基本維持しつつ加算拡張」を第一方針とする。
- 本ディレクトリは v1 設計の正本として運用する。

## 4. v1 スコープ

- ゲームルールの5種対応（Stud 系 + Flop 系）。
- テーブル進行ロジックの HORSE 対応。
- API/Realtime 契約の v1 拡張。
- DB モデルの拡張設計と移行方針。
- UI 表示・操作仕様の拡張設計。
- E2E シナリオ設計（正常系/異常系/境界値）。

## 5. v1 スコープ外

- No-Limit / Pot-Limit の導入。
- ユーザー卓作成機能の本実装。
- トーナメントモード。
- レーティング/ランキング本実装。
