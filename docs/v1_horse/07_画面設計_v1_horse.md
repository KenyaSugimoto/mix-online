# Mix Online 画面設計書（v1 HORSE）

> 作成日: 2026-02-13 ｜ 対象バージョン: v1 ｜ ステータス: Draft

## 1. 目的

HORSE 対応で UI が扱うべき表示・操作差分を定義する。

## 2. 画面一覧

- ログイン画面
- ロビー画面
- テーブル画面
- ハンド履歴一覧
- ハンド履歴詳細

## 3. ロビー画面

- 各卓カードに以下を表示:
  - 現在ゲーム（`gameType`）
  - ローテーション進捗（`handsSinceRotation / 6`）
  - 次ゲーム予告
- gameType ごとの視覚差分を定義（色・略称・ツールチップ）。

## 4. テーブル画面

### 4.1 共通

- プレイヤー席、スタック、アクション UI は共通。
- ゲーム種別ラベルを常時表示。

### 4.2 Stud 系表示

- 各プレイヤーの up/down card を表示。
- bring-in 局面をバッジ表示。

### 4.3 Flop 系表示

- 共通 board エリアを中央表示。
- HERO の hole card は常時表示、他者 hole は非表示。
- street 進行に応じて board を段階公開。

## 5. 履歴画面

- 一覧: gameType フィルタ追加。
- 詳細:
  - Stud 系: 各 street の個別配札/アクション
  - Flop 系: board 推移 + アクション
  - Hi/Lo 系: 片側不成立時の表示統一

## 6. エラー表示

- `INVALID_ACTION` は理由を UI 文言にマッピング。
- `AUTH_EXPIRED` は再ログイン導線へ遷移。

## 7. レスポンシブ方針

- モバイルでは board と席情報を縦積み。
- アクションボタンは固定フッター配置。

## 8. 実装チェック観点

- gameType 切替時に UI が安全に再描画される。
- 未知 enum を受け取ってもクラッシュしない。
- 再接続時 snapshot 適用で表示が復元される。
